My Project: Smart Home Air Quality Monitor

This project is a hardware-software integration focusing on immediate safety alerts. It is a **Smart Home Device** built on a Micro:bit shield, utilizing a **Gas Sensor (MQ-2)** and a **Dust Sensor** to detect environmental hazards and alert the user via a **Passive Buzzer** and **Onboard LED Blink**.

Core Functionality

The code is written in **MakeCode Blocks** (MicroPython equivalent) to simplify integration with the **Elecfreaks IoT:bit** shield. The system monitors two critical analog inputs and triggers a dual digital output (buzzer and LED) based on predefined safety thresholds.

  • **MQ-2 Gas Sensor:** Detects Butane, LPG, and Methane. The alarm is triggered when the gas reading exceeds a set threshold (e.g., **600** on the Micro:bit's 0-1023 analog scale).
  • **Dust Particle Sensor:** Monitors the concentration of airborne dust. The alarm is triggered when the reading exceeds a set safety threshold (e.g., **150 ug/m¬≥**).
  • **Passive Buzzer & Onboard LED:** Provides an immediate, simultaneous audible and visual warning for *either* hazard detection.

Final Code Logic (MakeCode Blocks)

// The Final MakeCode Block Logic Transcribed to Code

// --- On Start ---
let GAS_ALARM_THRESHOLD = 600
let DUST_ALARM_THRESHOLD = 150 
OLED.init(128, 64)
OLED.clear()

// --- Forever Loop ---
basic.forever(function () {
    // 1. Read Sensors
    let Gas_Reading = pins.analogReadPin(AnalogPin.P2) // MQ-2 on P2
    let Dust_Reading = Environment.readDust(DigitalPin.P13, AnalogPin.P1) // Dust Sensor on P13/P1
    
    // 2. Check Alarm Condition: TRUE if Gas OR Dust is too high
    let Alarm_On = (Gas_Reading > GAS_ALARM_THRESHOLD) || (Dust_Reading > DUST_ALARM_THRESHOLD)
    
    // 3. Display Data on OLED
    OLED.clear()
    OLED.showString("Dust(ug/m3):", 0, 0)
    OLED.showNumber(Dust_Reading, 90, 0)
    OLED.showString("Gas Reading:", 0, 20)
    OLED.showNumber(Gas_Reading, 90, 20)

    if (Alarm_On) {
        OLED.showString("--- ALARM ---", 0, 45)
    } else {
        OLED.showString("--- SAFE ---", 0, 45)
    }
    
    // 4. Buzzer & LED Alarm Action
    if (Alarm_On) {
        music.playTone(800, music.beat(BeatFraction.Quarter)) 
        basic.showLeds(`# . . . # . # . # . . . # . . # . # . # . . . #`)
    } else {
        music.rest(music.beat(BeatFraction.Sixteenth))
        basic.clearScreen()
    }
    basic.pause(2000)
})

My project development was a series of crucial learning moments that led me from complex theoretical integration to a focused, functional MVP. Every challenge taught me a key lesson about hardware communication, power management, and scope control.

Phase 1: The iOS App (Initial Failure)

I initially attempted to create a mobile app using Xcode that would wirelessly connect to the Micro:bit gas sensing device. While I completed the app's front-end, I encountered significant challenges connecting it to the hardware, repeatedly receiving a **"nonexit zero code error."**

**Key Learning:** Complex wireless communication (Bluetooth) between mobile devices and microcontrollers required specialized knowledge I needed more time to acquire. It taught me the value of simplifying the project scope.

Phase 2: The Keystudio Smart Home Model (Over-Scoping)

I pivoted to an all-encompassing Keystudio Smart Home model that included an OLED display, MQ-2 sensor, a fan, LEDs, revolving doors, a water sensor, and Bluetooth.

Image 3: The **functional Elecfreaks circuit** that became the core of the project, placed here to illustrate the complexity of the previous Keystudio attempts.

This approach led to **huge complications** due to battery lifespan issues and connection struggles (like the OLED display failing to turn on). I realized the project had too many features.

**Key Learning:** Feature creep and power management are critical challenges in hardware projects. This experience led me to focus strictly on the *core function*: gas and dust detection.

Phase 3: The Elecfreaks Circuit (Success)

I settled on the Elecfreaks circuit with a Micro:bit shield. I successfully applied lessons learned, focusing on a minimal viable product (MVP). I learned to program the Micro:bit effectively and confirmed that as long as I correctly reference the **analog pin** on the shield, connection is possible. I successfully integrated the MQ-2, the dust particle detector, and the buzzer. My final major challenge was **stabilizing the MQ-2 sensor readings**, which required additional research into sensor pre-heating and data averaging techniques.

**Outcome:** A functional, reliable safety device, proving that disciplined iteration leads to project completion.
