My Project: Smart Home Air Quality Monitor

This project is a hardware-software integration focusing on immediate safety alerts. It is a **Smart Home Device** built on a Micro:bit shield, utilizing a **Gas Sensor (MQ-2)** and a **Dust Sensor** to detect environmental hazards and alert the user via a **Passive Buzzer**.

üí° Project Details: Final MVP

Core Functionality

The code is written in **Python (MicroPython)**. The system monitors two critical analog inputs and triggers a digital output (the buzzer) based on predefined safety thresholds.

  • **MQ-2 Gas Sensor:** Detects Butane, LPG, and Methane. The alarm is triggered when the gas reading reaches or exceeds **400 parts per million (PPM)**.
  • **Dust Particle Sensor:** Monitors the concentration and size of airborne dust. The buzzer alerts the user when particles indicate **unsafe air quality** (e.g., in a heavily used workshop).
  • **Passive Buzzer:** Provides an immediate audible warning for either hazard detection.

Code Insight

// Conceptual Python Logic Snippet (Analog Read & Comparison)

from microbit import *
import music

GAS_THRESHOLD = 400 

def read_sensors_and_alert():
    gas_level = pin0.read_analog() # MQ-2 on P0
    dust_level = pin1.read_analog() # Dust Sensor on P1 (Conceptual)
    
    # Check for hazards
    if gas_level >= GAS_THRESHOLD or dust_level > DUST_THRESHOLD:
        music.play(music.BA_DING, wait=False)
    else:
        music.stop()

while True:
    read_sensors_and_alert()
    sleep(500)

My project development was a series of crucial learning moments that led me from complex theoretical integration to a focused, functional MVP. Every challenge taught me a key lesson about hardware communication, power management, and scope control.

Phase 1: The iOS App (Initial Failure)

I initially attempted to create a mobile app using Xcode that would wirelessly connect to the Micro:bit gas sensing device. While I completed the app's front-end, I encountered significant challenges connecting it to the hardware, repeatedly receiving a **"nonexit zero code error."**

**Key Learning:** Complex wireless communication (Bluetooth) between mobile devices and microcontrollers required specialized knowledge I needed more time to acquire. It taught me the value of simplifying the project scope.

Phase 2: The Keystudio Smart Home Model (Over-Scoping)

I pivoted to an all-encompassing Keystudio Smart Home model that included an OLED display, MQ-2 sensor, a fan, LEDs, revolving doors, a water sensor, and Bluetooth.

This approach led to **huge complications** due to battery lifespan issues and connection struggles (like the OLED display failing to turn on). I realized the project had too many features.

**Key Learning:** Feature creep and power management are critical challenges in hardware projects. This experience led me to focus strictly on the *core function*: gas and dust detection.

Phase 3: The Elecfreaks Circuit (Success)

I settled on the Elecfreaks circuit with a Micro:bit shield. I successfully applied lessons learned, focusing on a minimal viable product (MVP). I learned **Python** to program the Micro:bit and confirmed that as long as I correctly reference the **analog pin** on the shield, connection is possible.

I successfully integrated the MQ-2, the dust particle detector, and the buzzer. My final major challenge was **stabilizing the MQ-2 sensor readings**, which required additional research into sensor pre-heating and data averaging techniques.

**Outcome:** A functional, reliable safety device, proving that disciplined iteration leads to project completion.
